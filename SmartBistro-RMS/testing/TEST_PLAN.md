# Test Plan

## Testing Objective
The objective of testing is to confirm that SmartBistro works correctly, securely, and reliably for restaurant staff and managers.

## Test Cases

| Test ID | Feature | Test Scenario | Expected Result | Status |
|---|---|---|---|---|
| TC1 | Login | Staff enters valid login details | Dashboard opens | Pending |
| TC2 | Order creation | Waitstaff creates a new order | Order is saved successfully | Pending |
| TC3 | KDS | New order is submitted | Kitchen display updates | Pending |
| TC4 | Inventory | Menu item is sold | Inventory quantity decreases | Pending |
| TC5 | Reports | Manager opens sales report | Report is displayed | Pending |
| TC6 | Security | Invalid user attempts login | Access is denied | Pending |

## Quality Assurance Activities
- Peer review of documentation
- Code review through pull requests
- Test case execution
- Bug tracking through GitHub Issues
- Final user acceptance testing
