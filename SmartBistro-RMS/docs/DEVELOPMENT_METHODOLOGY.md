# Development Methodology and Project Management

## Agile Scrum Methodology

The SmartBistro Restaurant Management System adopts Agile Scrum because the project contains multiple connected modules, including digital ordering, Kitchen Display System, inventory management, reservation management, and analytics reporting. Agile Scrum supports incremental development and allows the team to adjust requirements based on feedback from stakeholders.

## Why Agile Is Suitable

SmartBistro is not a simple single-function application. It requires coordination between frontend interfaces, backend logic, database design, and testing activities. Scrum breaks the project into short sprints, allowing the team to deliver features progressively and identify problems early.

## Agile Compared with Waterfall

| Aspect | Agile Scrum | Waterfall |
|---|---|---|
| Development style | Iterative and incremental | Sequential |
| Requirement changes | Flexible | Difficult after planning |
| Testing | Continuous | Usually later in the project |
| Stakeholder feedback | Frequent | Limited |
| Risk control | Early issue detection | Issues may appear late |

## GitHub Version Control

GitHub is used as the main code repository and version control platform. It allows the team to store project files, track changes, create branches, raise issues, and review work through pull requests.

## Branching Strategy

| Branch | Purpose |
|---|---|
| main | Stable project files |
| documentation | Report and documentation updates |
| frontend-feature | Frontend interface work |
| backend-feature | Backend API work |
| testing-feature | Testing documentation and test cases |

## Project Management Tools

Trello or Jira can be used to manage the Agile workflow. Tasks are organised into To Do, In Progress, Testing, and Completed columns. This helps the team monitor progress and distribute work fairly.
