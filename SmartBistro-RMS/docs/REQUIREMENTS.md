# Requirements

## Functional Requirements

| ID | Requirement | Priority |
|---|---|---|
| FR1 | Users shall be able to create and manage digital orders. | Must Have |
| FR2 | Kitchen staff shall receive real-time order updates through KDS. | Must Have |
| FR3 | The system shall update inventory when menu items are sold. | Must Have |
| FR4 | Managers shall be able to view sales and inventory reports. | Should Have |
| FR5 | Customers shall be able to make table reservations. | Should Have |
| FR6 | The system shall support user login based on roles. | Must Have |

## Non-Functional Requirements

| ID | Requirement | Priority |
|---|---|---|
| NFR1 | The system shall respond quickly during peak restaurant hours. | Must Have |
| NFR2 | Customer and payment information shall be protected securely. | Must Have |
| NFR3 | The interface shall be simple and easy for staff to learn. | Should Have |
| NFR4 | The system shall be scalable for future branches. | Could Have |
| NFR5 | The system shall maintain high reliability during business hours. | Must Have |
