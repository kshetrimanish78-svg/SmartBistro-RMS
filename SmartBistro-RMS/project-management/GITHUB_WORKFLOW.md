# GitHub Workflow

## Repository Rules

1. The `main` branch should contain stable files only.
2. Team members should work in separate branches.
3. Each change should be saved using clear commit messages.
4. Pull requests should be reviewed before merging.
5. Issues should be used to track tasks and bugs.

## Example Commit Messages

- Added SmartBistro project overview
- Updated Agile methodology section
- Added database schema draft
- Created testing plan document
- Fixed documentation formatting
