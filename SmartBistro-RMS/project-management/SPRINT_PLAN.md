# Sprint Plan

| Sprint | Focus Area | Main Tasks | Output |
|---|---|---|---|
| Sprint 1 | Requirements and planning | Define scope, stakeholders, user stories, and backlog | Requirements document |
| Sprint 2 | Design | UI wireframes, database schema, system diagrams | Design artifacts |
| Sprint 3 | Ordering module | Create digital ordering workflow and basic frontend screens | Ordering prototype |
| Sprint 4 | KDS and inventory | Connect orders to kitchen display and stock updates | Integrated workflow |
| Sprint 5 | Testing and final review | Unit testing, integration testing, UAT, bug fixes | Final tested documentation |

## Sprint Review Process
At the end of each sprint, the team reviews completed work, identifies pending tasks, and adjusts the next sprint backlog.

## Sprint Retrospective
The team discusses what went well, what problems occurred, and what improvements should be made in the next sprint.
