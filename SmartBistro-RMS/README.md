# SmartBistro-RMS

SmartBistro is a Restaurant Management System designed for large urban restaurants. It supports digital ordering, kitchen display workflows, inventory monitoring, table management, and management reporting.

## Project Purpose
The purpose of this repository is to demonstrate the software engineering process used for the SmartBistro project, including requirements planning, Agile Scrum project management, version control, testing documentation, and system design evidence.

## Repository Structure

```text
SmartBistro-RMS/
├── docs/
├── frontend/
├── backend/
├── database/
├── testing/
├── designs/
└── project-management/
```

## Development Methodology
This project follows Agile Scrum. Work is divided into weekly sprints so that the team can develop, test, review, and improve the system incrementally.

## Main Modules
- Digital ordering system
- Kitchen Display System (KDS)
- Inventory management
- Table and reservation management
- Analytics and reporting

## Tools Used
- GitHub for version control
- Trello/Jira for task and sprint management
- Figma for UI prototype design
- Draw.io for system diagrams
- MySQL/PostgreSQL for database planning

## Team Workflow
The team uses feature branches, commits, issues, and pull requests to track project changes and support collaborative development.
